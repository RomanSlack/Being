Decoupled Talking Head Pipeline

ARTalk + SplattingAvatar — audio-driven Gaussian avatar rendering

Overview

Goal: Record 30 seconds of video, get a photorealistic avatar you can drive with any audio in real-time.

The pipeline is fully decoupled — the audio-to-motion model (ARTalk, SIGGRAPH Asia 2025) and the motion-to-render model (SplattingAvatar, CVPR 2024) are independent. Swap either piece without touching the other. Each component is state-of-the-art on its own.

  • Root cause identified: tracker uses asymmetric crop → squarefiy (pad to square) → resize

  • v4 training running with properly aligned images — PSNR already exceeding v3

Key Technical Insights

01
world_mat must store R.T (transposed). SplattingAvatar's getWorld2View2 transposes R internally, so the stored rotation must be R_opencv.T, not R_opencv.
02
IMavatar's FLAME uses a 4× geometry scale. All vertices, shape dirs, and pose dirs are multiplied by 4. Camera translation must be scaled by 4 to compensate.
03
Face crop must replicate the exact tracker pipeline: get_bbox (may clip at image edges → non-square) → squarefiy (pad symmetrically to square) → resize to 512. The padding shifts cx/cy away from center.
04
Expression coefficients transfer directly between ARTalk and SplattingAvatar — both use the FLAME 2020 expression space. First 50 of ARTalk's 100 dims map 1:1.
